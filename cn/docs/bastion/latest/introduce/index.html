一个堡垒机的实现，支持 SSH、Telnet、SVN、RDP、MySQL、PostgreSQL 等协议的访问。

简介 #

主要支持如下协议或者组件的代理功能：

  • 主机登录，支持 SSH、Telnet、VNC、RDP 等多种协议。
  • 数据库，支持 MySQL、PostgreSQL、Redis、Monogo 等。
  • 网络代理，允许通过 Socks5、http/https 协议进行访问。
  • K8S，支持常见的多数操作，包括 Docker 的实例。

另外，还提供了如下的基本功能：

  • 密罐，会记录多种协议的扫描，有效应对网络安全问题。
  • 审计，检查当前部署的服务配置是否存在安全风险。

对于商用的企业版本，支持多云部署以及其它企业级特性。

功能模块开源版本企业版本
审计日志支持记录事件到日志中同时支持记录会话的信息并进行回放
密钥保存简单的AES加解密支持复杂加解密，密钥轮转
禁止登录不支持可根据版本、客户端IP、按策略动态配置
AKSK不支持允许通过三方SDK方式访问REST API接口
用户管理不支持可以周期性的对登录主机用户进行轮转
权限管理支持单个角色支持多个角色的分权分域以及基于策略鉴权
超时设置包括登录、空闲、最大超时一致

其它特性。

  • 优雅退出，信号 INT TERM 会强制退出，而 USR1 则会等待用户优雅退出，此时可以再次启动服务。
  • 日志级别切换，启动时可以设置默认日志级别，通过 USR2 可以在 INFO 和 DEBUG 之间切换。

功能限制。

  • 只允许通过公私钥方式登录，不支持 Kerberos 或者 GSSAPI 模式，包括三方接口或者同步用户信息，因为这会导致依赖变多，系统更加复杂，这里尽量保持系统的独立和简单。

版本发布 #

当前通过 scripts/build 进行编译，通过 -tags integration 指定是集成环境。

----- GoLang
bash scripts/build

----- 安装相关依赖包
npm install
----- 运行开发模式
npm run dev
----- 发布打包
npm run build

后续会通过 Makefile 组织常用的命令。

说明
集成测试版本会注册测试用的 AK/SK 对，当前版本信息可以通过 bastion -v 命令查看，对应测试的 AK/SK 为 IntegrationTestAK01/IntegrationTestSecretKeyForSimpleTest01。

安装部署 #

一步步介绍如何初始化、安装、配置服务。

初始化数据库 #

包含了两类操作。

创建表结构 #

相关的数据库脚本保存在 scripts/database 目录下，按照版本进行归档，可以根据版本进行升级。

例如当前的服务器版本是 1.3.0，最新版本是 2.0.1，而在目录下存在 2.0.1 1.5.8 1.3.9 1.2.8 几个子目录里，因为并非每个发行版本都会涉及到数据库变更。那么升级到最新版本就需要按照 1.3.9 1.5.8 2.0.1 的顺序执行数据库相关的 SQL 文件。

也可以通过 bastion database 命令进行变更，此时会进行更多的检查、确认过程，防止误操作。

source scripts/database/0.1.0/bastion.cn.sql

初始化/Bootstrap #

注意，这里只用于初次安装。

在系统启动时会进行部分的初始化动作，初始化逻辑详见 Bootstrap 中的实现，主要包含如下内容：

  • 创建默认角色(user_role)，包括了 super supervisor admin auditor 。
  • 创建默认策略(DefaultAdminPolicy)，允许所有的执行操作。
  • 默认用户 admin 以及用户组 admin，该用户组会绑定 super/supervisor 角色以及上述的 admin 策略。
  • 主机 Key，主要用于 SSH 服务端，保存在 config 表中，对应 ssh_key_xxx_key。
  • 版本信息，用于后续的版本配置更新，同样保存在 config 表中，相见 Version。
  • 主机登录用户，名称为 default 使用 ed25519 的 SSH Key，用户名为 root 。

主要是由于部分的操作需要通过代码实现，无法直接通过 SQL 脚本生成，例如主机 Key 信息。

bin/bastion --level=debug --config=bin/etc/bastion.ini bootstrap

也可以通过如下命令查看 Bootstrap 是否成功。

bin/bastion --level=debug --config=bin/etc/bastion.ini bootstrap check

启动服务 #

根据不同的场景，提供了多种服务，可以通过如下方式启动。

Honey #

简单的 SSH 密罐系统，无论密码、公钥是什么值都会自动允许登录，只是会记录登录相关的信息。

----- 启动服务
bin/bastion --level=debug honey --config=bin/etc/honey.ini

----- 通过用户名密码以及证书登录
ssh -o PreferredAuthentications=password -o IdentitiesOnly=yes localhost -p 2880 -l root
ssh -o PreferredAuthentications=publickey -o IdentitiesOnly=yes -i id_rsa localhost -p 2880 -l root

其中配置项只有 LogPath、HostKeys、Listener 三个。

Bastion #

也就是最常用的跳板机，如下是最常用的参数。

bin/bastion --level=debug bastion --config=bin/etc/bastion.ini

初始化 #

当管理员创建用户时会生成一个 Token 字符串，用户第一次登录时以 invite:Token 作为用户名登录，而且需要指定 SSH Key ，这样就相当于 SSH Key 和用户进行了绑定，后续直接通过用户名和 SSH Key 登录即可。

系统第一次启动时 (Bootstrap) 会在日志中打印 admin 用户的 Token ，这样 admin 可以以同样方式进行登录，也可以从数据库的 user 表中查看，未进行加密。

----- 生成SSH Key证书
ssh-keygen -t rsa -f id_rsa_admin -N "YourPasswordRSA"
----- 指定上述的证书进行登录，后续会绑定到该证书
ssh -o PreferredAuthentications=publickey -o IdentitiesOnly=yes \
    -i id_rsa_admin localhost -p 9000 -l invite:RmfNjMuOk5OnKGbH
----- 后续就可以通过如下命令登录了
ssh -o PreferredAuthentications=publickey -o IdentitiesOnly=yes \
    -i id_rsa_admin localhost -p 9000 -l admin

另外，出于安全考虑，上面设置的密钥对的密码信息，每次登录都需要输入密码才行，可以通过如下方式记录密钥，一次登录后就无需再输入密码了。

eval $(ssh-agent)
ssh-add id_rsa_admin         --> YourPasswordRSA
ssh-add -l

注意，现在是无法登录主机的，需要添加主机、主机组以及用户组之后，才可以，更多内容查看 权限管理。

选择主机 #

主机存在如下的绑定关系 Host --n:n-> HostGroup HostGroup --n:n-> UserGroup，在选择需要登录的主机时，会按照 UserGroup->HostGroup->Host->Login 顺序选择，如果只有一个，那么就不会提示选择。

其它 #

命令行 #

健康检查，会通过 healthcheck 用户进行登录，对应如下命令。

bin/bastion healthcheck --addr=127.0.0.1:2880 --wait=true

还支持 tools 子命令，用作一些基本工具，其子命令包括了：

  • encrypt/decrypt 通过 --cipher 指定加解密的配置参数。
  • config 打印配置信息，默认打印全部参数，可以指定某个模块。

如下是常用的示例。

------ 加密/解密指定文本
bin/bastion tools --cipher=aes:YourSecretKey encrypt YourPlanText
bin/bastion tools --cipher=aes:YourSecretKey decrypt '2$0$Avqruz6IAefMLJDC5Zn4t1xF+PNE9KL81MVGnA'

------ 打印所有配置
bin/bastion --level=debug --config=bin/etc/bastion.ini tools config dump
------ 打印指定的配置
bin/bastion --level=debug --config=bin/etc/bastion.ini tools config dump ssh
