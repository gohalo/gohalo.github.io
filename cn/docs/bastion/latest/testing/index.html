环境准备 #

这里提供了基于 Docker 或者 K8S 相关的测试环境，在源码的 scripts/docker 目录下有对应 Dockerfile 文件可以创建测试所需的镜像，当前包括了 SSH 和 Telnet 两类，而 SSH 还包含不同认证方式。

----- 环境准备，查看镜像信息，清理历史容器
docker image ls
docker ps -a | grep "Exited" | awk '{print $1 }' | xargs docker rm
----- 启动两个SSH、一个Telnet，常用Docker操作
docker run -p 127.0.9.10:5022:22 -d --name sshd001 -P sshd:latest
docker run -p 127.0.19.1:5022:22 -d --name sshd002 -P sshd:latest
docker run -p 127.0.29.1:5023:23 -d --name telnetd001 -P telnetd:latest
----- 也可以直接通过compose启动
docker -f compose.yaml compose up -d

启动简单的 K8S 环境可以参考 K8S简单部署 中的介绍。

----- 查看Nodes和Pods信息
kubectl get nodes -o wide
kubectl get pod --all-namespaces -o wide
----- 启动服务
kubectl run ssh00 --image=mysshd:v1 --image-pull-policy=Never --port=22 --hostport=2020
kubectl run ssh01 --image=mysshd:v1 --image-pull-policy=Never --port=22 --hostport=2021
kubectl run telnet01 --image=mytelnetd:v1 --image-pull-policy=Never --port=23 --hostport=3100
----- 查看、登录、删除服务
kubectl exec telnet001 -i --tty -- /bin/sh
kubectl describe pods telnet001
kubectl delete pods --all

----- 访问本地服务器 opsadmin:admin root:admin
telnet 192.168.1.4 3100
ssh -o PreferredAuthentications=password localhost -p 2020 -l root

准备数据库 #

----- 启动数据库
/opt/MySQL/home/msb_8_0_27/start
----- 准备数据库环境
/opt/MySQL/binary/8.0.27/bin/mysql -u msandbox -P 8027 -h 127.0.0.1 -pmsandbox

然后可以根据需要初始化数据库，以及 Bootstrap 等操作。

测试用例 #

在 scripts/testing 目录下包含相关的测试脚本，其中，utils.py 通过 FastAPI 封装了常见的基础 API 操作，默认通过 AK/SK 进行访问。

University #

主要是针对具体的模拟场景，例如 university 大学，如下是常见的 API 测试脚本：

  • 0-bootstrap.py 获取并检查内置信息，包括 admin 用户/用户组，默认主机登录信息 default 等。
  • 1-prepare.py 准备测试环境，包括主机登录信息、主机组、主机、用户组、用户、策略绑定等，同时会绑定 SSH 的私钥，私钥可以通过 generate.sh 脚本生成。
  • 2-ssh.py 用于测试登录 SSH 实现，可以多次调用。
  • 3-aksk.py 当前主要用于查看 AKSK 信息。
  • 3-hostgroup.py 测试 HostGroup 使用，添加之后立即删除，仅做测试使用。
  • 3-usergroup.py 测试 UserGroup 使用，同上，添加之后立即删除，仅做测试使用。
  • 3-hostlogin.py 主机登录信息配置。

登录相关 #

上述的 1-prepare.py 脚本会配置一些用户、用户组、主机组信息，详细可以查看脚本，如下可以作为测试。

先保存相关的密码，减少后续的密码输入。

eval $(ssh-agent)
ssh-add scripts/testing/university/keys/id_rsa_admin         --> YourPasswordRSA
ssh-add scripts/testing/university/keys/id_rsa_j00128901     --> YourPasswordRSA
ssh-add scripts/testing/university/keys/id_rsa_m00228901     --> YourPasswordRSA
ssh-add scripts/testing/university/keys/id_ed25519_r00258912 --> YourPasswordED25519
ssh-add scripts/testing/university/keys/id_ed25519_f10000102 --> YourPasswordED25519
ssh-add -l

然后可以登录到命令行。

ssh -o "SendEnv LANG" -o PreferredAuthentications=publickey -o IdentitiesOnly=yes \
    -i scripts/testing/university/keys/id_rsa_admin localhost -p 9000 -l admin

ssh -o "SendEnv LANG" -o PreferredAuthentications=publickey -o IdentitiesOnly=yes \
    -i scripts/testing/university/keys/id_rsa_m00228901 localhost -p 9000 -l invite:RmfNjMuOk5OnKGbH
ssh -o "SendEnv LANG" -o PreferredAuthentications=publickey -o IdentitiesOnly=yes \
    -i scripts/testing/university/keys/id_rsa_m00228901 localhost -p 9000 -l m00228901
ssh -o "SendEnv LANG" -o PreferredAuthentications=publickey -o IdentitiesOnly=yes \
    -i scripts/testing/university/keys/id_ed25519_r00258912 localhost -p 9000 -l r00258912

或者直接登录主机。

ssh -o PreferredAuthentications=publickey -i ~/.ssh/id_test_rsa localhost -p 2880 -l host:rds.console01root
ssh -N -f -L 5600:127.0.0.1:8027 -o PreferredAuthentications=publickey -i scripts/testing/keys/id_rsa01 localhost -p 2880 -l m00228901

与 K8S 采用兼容的协议，第一个字节的 0 1 2 分别表示 stdin stdout stderr 。
