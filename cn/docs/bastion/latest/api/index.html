简介 #

通过 HTTP 访问，访问时鉴权当前有两种方式：A) AK/SK 通常用于机机访问；B) Token 用于页面的访问。

常见错误：

  • 401 没有授权信息，默认采用 Bear Header，也可能格式错误。

另外，在服务端会检查 Token 的过期时间

命名规范 #

对于字段会配置命名约束的检查，如下列举部分。

Name    --> [a-zA-Z0-9_]{4,63}
Config  --> {0,1024}
Comment --> {0,255}

HostLogin #

主机的登录信息，可以是用户名/密码、证书。

----- 添加 [super, admin]
POST /host/login
{
    "name": "PracticeRootKey",    # [name] 主机登录名称，唯一索引
    "type": "key",                # 登录类型，支持 key 证书；password 密码
    "schema": "all",              # 允许使用的 scheam 类型，all 表示所有，其它如 ssh telnet k8s 等
    "login": "root",              # [name] 登录用户名
    "login_type": "admin",        # 用户类型，包括 admin 管理用户、normal 普通用户
    "key_type": "ed25519",        # key 证书类型
    "key_lenght": 4096,           # key 证书长度
    "passphrase": "YourPassword", # 对于 key 是私钥的密钥，而 password 是密码   ----> 待定
    "comment": "Root password",   # 注解
    "config": {                   # [config] 配置相关
        "steps": [{               # 自动登录相关
            "expect": "login:",
            "send": "{{user}}",
        }, {
            "expect": "Password:",
            "send": "{{password}}",
        }, {
            "expect": "$",
            "failed": "Login incorrect",
        }]
    },
}

----- 获取单个/多个主机登录信息 [super, admin]
GET /host/login?id=xxx&name=xxx&decrypt=true
GET /host/login?marker=10,limit=20,desc=true,sort=id
GET /host/login?offset=0,limit=20,desc=true,sort=id,total=true

----- 通过ID或者Name删除，注意ID也是字符串 [super, admin]
DELETE /host/login
["PracticeRootKey", "12"]

----- 修改指定的字段 TODO
PATCH /host/login
[
]

这里的用户类型用于后续的维护，只有 admin 用户可以用来修改其它用户，例如删除用户、修改密码等。

Host & HostGroup #

----- 添加主机组 [super]
POST /host/group
{
    "name": "Practice",             # [name] 主机组名称
    "type": "flat",                 # 主机组类型，flat tree
    "comment": "Computer Science",  # [comment] 注释
    "host_login": [                 # 绑定登录信息
        "PracticeRootKey", "PracticeRootPassword", "PracticeTelnetKey"
    ],
}

----- 获取主机组信息
GET /host/group?id=xxx&name=xxx
GET /host/group?marker=10,limit=20,desc=true,sort=id
GET /host/group?offset=0,limit=20,desc=true,sort=id,total=true

----- 通过ID或者Name删除，同上，ID也是字符串 [super]
DELETE /host/group
["PracticeRootKey", "12"]

----- 添加主机 [super]
POST /host
{
    "name": "PracticeSSH001",       # [name] 主机别名
    "host": "127.0.0.1:2020",       # 地址
    "schema": "ssh",                # 类型 ssh telnet k8s
    "comment": "SSH Host",          # [comment] 注释
    "host_group": ["Practice"],     # 指定主机组
}

----- 获取主机信息
GET /host?id=xxx&name=xxx
GET /host?marker=10,limit=20,desc=true,sort=id
GET /host?offset=0,limit=20,desc=true,sort=id,total=true

----- 通过ID或者Name删除，同上，ID也是字符串 [super]
DELETE /host
["PracticeSSH001", "12"]

User & UserGroup #

----- 添加用户组 [super]
POST /user/group
{
    "name": "ComputerScience2021",    # [name] 用户组名
    "login": "root",                  # [name] 该用户组默认的登录用户
    "login_policy": "Unspecified",
    "comment": "2021 Class",          # [comment] 注释
    "host_group": ["Practice"],       # 绑定主机组
}

----- 获取用户组信息
GET /user/group?id=xxx&name=xxx
GET /user/group?marker=10,limit=20,desc=true,sort=id
GET /user/group?offset=0,limit=20,desc=true,sort=id,total=true

----- 通过ID或者Name删除，同上，ID也是字符串 [super]
DELETE /user/group
["ComputerScience2021", "12"]


----- 添加单个用户，允许绑定用户组，如果已经存在则更新
POST /user
{
    "uid": "r00258912",                    # [name] 用户ID
    "name": "Richard",                     # [name] 用户名
	"phone": "+8612345678901",             # 用户手机号
    "email": "test@foobar.com",            # 用户邮箱地址
	"password": "YourPassword",            # 登录密码
	"login": "root",                       # 默认登录主机用的用户名
    "random": false,                       # 如果密码为空且该值为 true 则自动生成密码
    "invite": true,                        # 是否通过 invite 方式注册
    "config": "",                          # 用户侧的配置
    "comment": "Your Comment",             # 注释
    "user_group": ["ComputerScience2021"]  # 绑定的用户组
}

----- 获取所有用户信息
GET /user?limit=10&marker=120
----- 获取单个用户信息，可以根据ID或者UID查找，可以选择是否查询相关的用户组
GET /user?id=xxx&uid=xxx&email=xxx&phone=xxx&detail=true&with_role=true&with_policy=true

----- 通过ID或者Name删除，同上，ID也是字符串 [super]
DELETE /user
["r00258912", "12"]


----- 添加用户策略
POST /user/policy
{
  "name": "Test_PAP_0713",                 # [name] 用于显示
  "version": "1.1",                        # [semver] 策略的版本信息
  "priority": 10,                          # [0-9999] 策略的优先级配置
  "effect": "allow",                       # [allow/deny] 允许还是拒绝
  "description": "example",                # [comment] 描述详细信息
  "action": [                              # 支持的动作
    "Bastion:*:*"
  ],
  "resource": [                            # 管理的资源
    "Bastion:*:*:*:*",
  ],
  "condition": {                           # 配置条件
    "Bool": {
      "g:MFAPresent": ["true"]
    },
    "StringEquals": {
      "g:Username": ["abc", "aaa"],
      "obs:BucketName": ["bucketName1", "bucketName2"]
    },
    "IsNullOrEmpty": {
      "g:UserId": null
    }
  },
  "user_group": ["ComputerScience2021"]    # 绑定的用户组
}

----- 获取所有用户信息
GET /user?limit=10&marker=120
----- 获取单个用户信息，可以根据ID或者UID查找，可以选择是否查询相关的用户组
GET /user?id=xxx&uid=xxx&email=xxx&phone=xxx&detail=true

----- 通过ID或者Name删除，同上，ID也是字符串 [super]
DELETE /user
["r00258912", "12"]


----- 用户登录
POST /user/login
{
    "uid": "b10000178",
    "password": "YourPassword"
}
===> resp
{
    "uid": "b10000178",
    "name": "Bruce",
    "token": "JWT Token",
    "expire": 1716127247
}

----- 获取当前用户信息
GET /user/current

注意，这里用户的密码只做单向认证，不支持解密操作，另外，创建用户的时候会返回明文密码，作为初始密码。

其它 #

AKSK #

用于应用访问 API 接口，注意，只有首次添加时会返回 SK 信息，所以，页面上需要确保用户能够保存。

----- 添加 [super]
POST /aksk
{
    "name": "YourAKSKName",    # [Name] 名称
    "appid": "bastion",        # [Name] 应用名称，可以用于后续的封堵
    "comment": "AKSK for Bastion application",   # [Comment] 注解
}

----- 获取AKSK信息 [super]
GET /aksk?ak=xxx&id=true
GET /aksk?size=10&marker=120

----- 通过ID或者Name删除，同上，ID也是字符串 [super]
DELETE /aksk
["YourAKSKName", "12"]
