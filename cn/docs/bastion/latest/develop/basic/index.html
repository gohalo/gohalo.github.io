简介 #

GoLang 基础库 golang.org/x/crypto/ssh 仅提供了基本 SSH 能力，其中 gliderlabs/ssh 是不错的服务端封装，包括更上层的 charmbracelet/wish 封装及其插件，这里大量参考该库，而 goph 是客户端封装。另外，还有些不错的 SSH 相关的工具，可以参考 gotty、wetty 等。

命令行以及 Shell 的解析通过 urfave 实现，源码可以参考 gitHub 实现。

同时在 cmd/examples 目录下包含了很多测试代码。

  • golang.org/x/term 修改为 pkg/xterm 增加 SetHistory() 函数的实现。在列表选择时，可以不将数字保存到历史中，防止历史信息被污染。

各个模块之间有相互依赖关系，这里简单梳理如下。

上下文 #

通常是通过 context.Context 来实现上下文的传递，除了作为定时器、退出等场景，最常用的是携带值，不过其实现的逻辑类似于线性链表，所以，通常希望不要携带太多的值，或者说不要嵌套太多层数。

在实现时，尽量能通过某个结构体包含所有的信息，对于 SSH 来说，就是每个 Channel 为一个上下文，这个上下文中会尽量包括多的数据，获取到结构体之后可以直接修改对应的值。

Pipe #

负责客户端和服务端之间的请求转发，这是 Bastion 的核心处理逻辑。

Mux #

用作从客户端到 Shell/服务端 的请求转发，只有在通过 Shell 登录时才会涉及，如果是直接登录则不会存在 Mux 做中间转发，会有单独协程从左侧读取然后将数据发送到 Shell 或者 服务端 。

配置相关 #

采用 INI 格式的配置文件，默认就是全局配置。

LogPath=/var/log/test.log
[auth]
Region=test
Method=role

其中全局参数大部分在 internal/bastion/config.go 中实现，不同模块可以在 init() 函数中通过 RegisterItemConfig() 或者 RegisterDomainConfig() 注册钩子函数。

ed25519 #

当前 GoLang 的 SSH 库支持从 OpenSSH 格式读取，但是无法生成 OpenSSH 格式内容，而且 ed25519 只支持 OpenSSH 格式，所以需要独立实现一套 ed25519 的 OpenSSH 生成方案，相见 kite/openssh 中的实现。

详细可以参考如下的仓库，而且还提供了 DSA 的私钥支持。

https://github.com/ScaleFT/sshkeys
https://github.com/mikesmitty/edkey

gorm 关联 #

通过 gorm 的关联操作默认应该是无需更新 Role 表的，但是实际上下面的 SQL 仍然会更新。

package main

import (
	"log"

	"gorm.io/driver/mysql"
	"gorm.io/gorm"
	dblog "gorm.io/gorm/logger"
)

type Role struct {
	ID   int
	Name string
}

func (*Role) TableName() string {
	return "roles"
}

type User struct {
	ID     int
	Name   string
	Age    int
	Gender string
	Roles  []*Role `gorm:"many2many:map_user_role"`
}

func (*User) TableName() string {
	return "users"
}

func main() {
	dsn := "root:msandbox@tcp(127.0.0.1:8027)/test?charset=utf8&parseTime=True&loc=Local"
	db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{
		Logger: dblog.Default.LogMode(dblog.Info),
	})
	if err != nil {
		log.Printf("Open database failed, %v.", err)
		return
	}

	var user User
	if err := db.First(&user, "name = ?", "andy").Error; err != nil {
		return
	}

	var role []Role
	if err := db.Find(&role, "name IN (?)", []string{"admin"}).Error; err != nil {
		return
	}
	if err := db.Model(&user).Association("Roles").Replace(role); err != nil {
		return
	}
}

表结构 #

  • user 用户信息，包括了 uid phone invite 等信息。
  • host_login 用于登录主机，会包含公/私钥以及私钥的密码(含公钥 sha256 值)，或者登录用密码。
  • user_login_key 保存了用户的证书信息，主要是公钥信息以及公钥的 sha256 值，注意，不会保存私钥信息，通过 user_id 与 user 表中的用户关联。
