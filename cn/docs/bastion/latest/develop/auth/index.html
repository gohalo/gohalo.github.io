密钥管理 #

主要是用于加解密相关，其中部分场景需要进行解密，例如 SSH 登录密码的保存，登录前需要先解密；还有就是用户的登录密钥，因为只需要对密码进行验证，为了保证密码的安全性，采用单项加密算法。

管理的密码除了算法还有版本信息，主要是为了方便进行迭代，最常见的是对称加解密的加密密钥修改，还有就是部分加密参数的调整，这些信息通过 Suites 维护，可看作是 CipherSuites 简写。

在 配置文件 中可以参考配置。

实现 #

通过 type Suites struct 保存相关的信息，默认会创建一个全局的对象，同时提供了类似 Verify() Encrypt() Decrypt() 等全局函数。

加密后的信息类似 2$0$Avqruz6IAefMLJDC5Zn4t1xF+PNE9KL81MVGnA，分别为 加密算法 加密版本 加密信息 。

权限管理 #

登录方式包含了 SSH-Key SSH-Password HTTP-Token HTTP-AKSK 四种方式，代码中通过 auth.AuthContext 保存鉴权相关信息，会通过 NewAuth() 函数根据类型创建，可以直接搜索该函数。

在 NewAuth() 只会通过登录信息初始化，不会执行数据库查询等动作，不同的登录方式。

type AuthInfo struct {
	Type       *AuthType   // 用户登录的类型，包括了ssh-key ssh-password http-token http-aksk
	Credential interface{} // 不同类型保存了不同认证信息，无需查询数据库
}

type AuthContext struct {
	Info     *AuthInfo       // 不同登录方式对应的初始信息不同
	User     *model.User     // 只有SSH登录后会根据证书指纹更新，相见 handleSSHKeyAuth()
	Roles    map[string]bool // 有些鉴权会自动保存角色，例如 token，有些需要从数据库中读取
	Policies []warden.Policy
	Context  warden.Context
}

其中 AKSK 允许所有操作，其它请求会通过 AuthContext.RefreshWithID/UID() 函数进行刷新，最终通过 api.GetUser() 获取用户信息，根据配置包含 Roles、Policies 信息，然后通过 User.GetRoleMap() 以及 User.GetPolicyList() 获取最终的角色和权限信息。

同时会通过 UserContext.WithUser() 将角色、策略信息添加到用户上下文中。

鉴权 #

通过 AuthContext.IsAllowed() 实现，默认会先检查 Policies 然后是 Roles 。

策略实现 #

参考了 Ladon 实现，从数据库中读取后，然后进行解析，相见 User.GetPolicyList() 实现，最终对应到 internal/model/user_policy.go 代码。

登录 #

登录过程会在 handleSSHKeyAuth() 函数中通过 api.BindUserKey() 绑定，登录时根据 SSH Key 的指纹获取 UserLoginKey 对象，然后再查询到用户信息。
