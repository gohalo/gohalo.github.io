错误处理 #

有如下的几个要求：

  • 区分类型，用来统计当前系统是否健康，有那些子系统报错，例如用户错误、数据库错误。
  • 错误信息，处于安全考虑部分详细信息不会展示给客户，仅仅会在系统日志中打印，当然，有些是一致的。
  • 国际化，Linux 中通过 LANG 环境变量设置，会处理 HTTP 请求、命令行等。

实现保存在 cargo.com/kite/errors 库中，包含了大部分相关的字段，而具体的错误定义在 api 中，使用时，在 api 中不会打印错误日志信息，只会返回错误。

Equal() 判断错误是否相同

String()  包含详细信息
UserErr() 返回给用户错误

基础库 #

采用 GoLang 最基础的错误处理，也就是常规的 errors.New() fmt.Errorf() 使用，也包括基于 fmt.Errorf("%w") 这种方式，同时为了方便识别错误，在错误信息中会添加模块前缀。

更多内容可以参考 pkg/secret 包的实现，错误信息会添加 secret: xxx 前缀，为了区分不同算法还可以是类似 secret(aes) 这种前缀。

数据库 #

提供了 HandleDBError 对数据库的错误进行封装，例如，对于 MySQL 可以通过返回码判断错误类型。

if ori, err := HandleDBError(db.Omit("CreateAt", "UpdateAt").Create(group).Error, "user group"); err != nil {
    if ignoreExists && ErrItemAlreadyExist.Equal(err) {
        return group, nil
    }
    logger.Error("%vCreate user group '%v' failed, %v.", trace, group, ori)
    return nil, err
}

后续可以进一步抽象处理，例如 CreateItem 函数。

HTTP #

通过 echo.HTTPError 结构体定义，常用方法如下。

err := echo.NewHTTPError(http.StatusBadRequest) // 使用 http.StatusText 内容
err := echo.NewHTTPError(http.StatusBadRequest, "invalid auth info")
err.SetInternal(io.EOF) // 设置内置错误信息

SSH 相关 #

登录 SSH 服务时，如果鉴权失败只会显示 Permission denied 报错，没有具体的详细信息，所以，在该阶段尽量简单，只获取相关的鉴权信息。

日志 #

通过对 go.uber.org/zap 库进行封装，支持 echo gorm 等三方库的日志接口，同时提供常规日志、审计日志、访问日志等场景。

其中 echo 要求的 Logger 接口比较多，目前暂不确定很多接口的使用场景，只实现了部分。
