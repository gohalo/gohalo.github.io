简介 #

权限配置策略针对的是用户组以及主机组，也就是说用户具有的权限必须要添加到用户组之后才可以，支持基于角色和策略的授权方式。

角色 #

默认包含如下的角色。

  • 普通用户：登录、查看所在用户组、关联的主机组，可以配置允许的登录类型。
  • 用户组管理员(admin)：可以添加、删除、冻结、解冻用户，维护主机组内机器。
  • 用户组审计人员(auditor)：只能查看用户组历史记录，无法登录、修改配置等。
  • 超级管理员(super)：所有权限，用来创建用户组、主机组等。
  • 超级审计员(supervisor)： 可以查看所有用户组历史记录，无法登录、修改配置等。

普通用户，可以用于登录主机、查询数据库等常规操作，但是默认只具有只读权限，其它操作需要进一步申请权限。

分权分域两个比较关键的点：A) 用户管理，用户组管理员可以禁用用户；B) 主机管理，用户组管理员可以删除但是无法添加。

策略 #

这里列举了当前支持的功能以及后续规划的功能，希望可以提供一个便于配置的安全策略机制，安全策略与用户组绑定，不同用户组之间的优先级顺序同样有效。

其中 Action 通过 ServiceName:ResourceType:Operation 表示。

----- 主机信息
Bastion:Host:Get        Host:ID
Bastion:Host:List
Bastion:Host:Add                  Super Admin
Bastion:Host:Delete               Super Admin

----- 主机组信息
Bastion:HostGroup:Get        HostGroup:ID
Bastion:HostGroup:List
Bastion:HostGroup:Add                       Super
Bastion:HostGroup:Delete                    Super

----- 主机登录信息
Bastion:HostLogin:Get        HostLogin:ID
Bastion:HostLogin:List
Bastion:HostLogin:Add                       Super Admin
Bastion:HostLogin:Delete                    Super Admin

----- 用户信息
Bastion:User:Get         User:ID
Bastion:User:List
Bastion:User:Add                            Super
Bastion:User:Delete                            Super

----- 用户组信息
Bastion:UserGroup:Get         UserGroup:ID
Bastion:UserGroup:List
Bastion:UserGroup:Add                            Super
Bastion:UserGroup:Delete                            Super
Bastion:UserGroup:Bind                              Super Admin

----- 策略信息
Bastion:UserPolicy:Get         UserGroup:ID
Bastion:UserPolicy:List
Bastion:UserPolicy:Add                            Super
Bastion:UserPolicy:Delete                            Super

----- AKSK 操作 Super
Bastion:AKSK:Get        AKSK:ID/AK
Bastion:AKSK:List
Bastion:AKSK:Add        AKSK:ApplicationName
Bastion:AKSK:Delete

----- 系统管理权限
Bastion:Admin:Encrypt    对明文数据进行加密
Bastion:Admin:Decrypt    对明文数据进行解密
Bastion:Admin:RereshKey  对其它用户的SSHKey进行更新
Bastion:Admin:List       查看系统监控参数
Bastion:Admin:Modify     查看、修改系统参数



----- 用户(User)、用户组(UserGroup)、主机(Host)、主机组(HostGroup)
Bastion:User:Add       添加用户，也包含绑定到用户组
Bastion:User:List      查看用户以及关联用户组
Bastion:User:Modify    修改用户属性
Bastion:User:Delete    删除用户，包括用户以及绑定信息
Bastion:UserGroup:Bind 用户组绑定用户



----- 系统管理权限
Bastion:Admin:List     查看系统监控参数
Bastion:Admin:Modify   查看、修改系统参数

----- 主机和主机组
Bastion:SSH:List    查看绑定主机组以及组内的主机 manager
Bastion:SSH:Modify  可以添加、修改、删除绑定主机组以及组内的主机，也包括主机关联的账号 manager
Bastion:SSH:ListAll 查看所有信息，包括租户、主机组、用户组、账号等等 admin
Bastion:SSH:Login   主机登录，查看所有可以登录主机、账号，会通过上下文判断是否进行黑白名单限制
Bastion:SSH:Files   文件操作，包括了上传、下载等，支持二次审批

----- MySQL数据库相关权限
Bastion:MySQL:DDL
Bastion:MySQL:DML
Bastion:MySQL:DCL
Bastion:MySQL:Query

另外，还有 Resource 相关，通过 ServiceName:Region:AccountID:ResourceType:ResourcePath 表示，其中后两者包括了。

AKSK:ID/Name  --> Create

策略类似如下。

{
  "description": "One policy to rule them all.",
  "subjects": ["users:<peter|ken>", "users:maria", "groups:admins"],
  "actions" : ["delete", "<create|update>"],
  "effect": "allow",
  "resources": [  // 配置用户组
    "Bastion:Host-*:.*"
  ],
  "conditions": {
    "remoteIP": {
        "type": "CIDRCondition",
        "options": {
            "cidr": "192.168.0.1/16"
        }
    }
  }
}

  "conditions": {
    "CIDR": {
        "RemoteIP": ["192.168.0.1/16"]
    },
    "DateBetween": {
        "Date": "192.168.0.1/16"
        "RemoteIP": "192.168.0.1/16"
    }
  }


{
  "name": "Test_PAP_0713",
  "description": "example",
  "version": "v1.0.0",
  "effect": "Allow",
  "action": [
    "obs:bucket:create"
  ],
  "resource": [
    "obs:*:*:Bucket:mybucket/test1/*",
    "obs:*:*:Bucket:mybucket/test2/aaa"
  ],
  "condition": {
    "Bool": {
      "g:MFAPresent": ["true"]
    },
    "StringEquals": {
      "g:Username": ["abc", "aaa"],
      "obs:BucketName": ["bucketName1", "bucketName2"]
    },
    "IsNullOrEmpty": {
      "g:UserId": null
    }
  }
}
