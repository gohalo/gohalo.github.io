简介 #

在 ssh 子模块中保存相关配置。

主机证书 #

SSH 客户端登录时会验证主机公钥信息，如果是部署了多台机器，那么各个机器上的私钥信息最好保持一致，或者直接保存在数据库中维护。

HostKeys=file:bin/ssh_host_rsa_key,file:/etc/bastion/ssh_host_ecda_key,db:ssh_host_ed25519_key

支持文件和数据库保存，文件可以是相对路径或者绝对路径，文件不允许加密，保存在数据库中的证书可以加密。

默认配置为 file:/etc/cargo/bastion/ssh_host_ed25519_key 。

登录策略 #

主要是为了让用户更好的排查错误，可以通过 LoginPolicy 配置项修改，支持如下几种方式：

  • bypass 认证阶段直接成功，在接受请求后处理；
  • basic 默认值，标准的认证流程，直接在鉴权阶段处理；
  • smart 因为 password 只支持健康检查，这里 password 方式在认证阶段处理，而 publickey 方式则接受请求后处理。

注意，选择了 bypass 之后，只有一种登录方式，而 basic 则可以在某种方式失败后尝试其它方式。
