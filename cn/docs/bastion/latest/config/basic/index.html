简介 #

核心的包括了配置文件以及不同用户、用户组的配置，这里重点介绍配置文件的核心配置内容。

默认的配置文件保存在 /etc/cargo/bastion/bastion.ini 可以在启动时通过 --config=/Your/Config/File.ini 方式指定，可以通过 # 添加注释(包括整行和行尾)，如下是常见的配置项。

另外，为了方便调试配置，可以在启动进程时通过 --level=debug 参数指定打印读取配置文件时的调试信息，而实际的日志级别通过配置文件中的 LogLevel 配置。

其中部分配置项支持多次配置，例如 Cipher 的配置项可以为 Cipher00 Cipher01 等。

日志配置 #

可以配置应用启动之后的参数，启动过程中可以通过命令行的 --level 参数配置。

# 日志级别，支持 debug info warn 等
LogLevel = info
LogRotate = 50,4,30,true
LogBufferSize = 16
LogPath = /var/log/cargo/bastion/bastion.log
RecordBasePath = /var/log/cargo/bastion/record

其中 LogLevel 和 LogPath 分别用于配置日志级别以及保存路径，通过 LogBufferSize 设置缓存大小(单位是 KB)，默认为 0 也就是不缓存，而 RecordBasePath 配置会话录制文件保存路径的基础路径，另外，启动后可以通过 kill -SIGUSR2 <PID> 切换日志级别。

而 LogRotate 用来配置日志切割，包括了四个字段，通过 , 分割，分别为：A) 日志大小 (MB)；B) 保留备份文件数；C) 最长保留时间 (天)；D) 是否压缩 (否)。

加解密 #

Cipher=aes:xxxxxx
CipherConf=conf@aes:0,host@aes:1

通过 Cipher 配置支持的加密算法，使用分号 : 分割，分别表示对称加密算法类型、版本信息、密钥参数(通过逗号 . 分割)，允许指定多次，不同场景下参数可以简化，例如：

  • 只需要加密算法，没有参数或者采用默认，版本默认为 0，如 plain md5 pbkdf2 等。
  • 不指定版本，只指定参数，如 aes:YourRootSecret pbkdf2:2.8.2000.sha256 等。
  • 同时指定版本，用于后续的代码论转，通常只有对称加密算法需要。

不同加密算法对应的参数不同，例如对于 AES 来说就是简单的 Key 即可，其中加密算法支持如下几种：

  • Plain 不加密，仅用于调试，不建议线上使用。
  • AES 对称加密，用于需要解密的场景，必须要指定根密钥参数 (需加固)。
  • MD5 SHA256 SHA512 单向加密，容易通过彩虹表破解，不建议使用。
  • PBKDF2 单向加密，参数为 salt_min.salt_max.round.hash_func， 可选，默认为 4.8.1000.sha256。
  • Bcrypt 单向加密，只需要单个成本参数，默认为 10，取值范围 [4, 31]。

注意，对于 PBKDF2 来说，不同参数取值范围为 Salt[2, 32] Round[10, 3000]，而 HashFunc 当前只支持 sha256 一种，后续会进行扩展。

通过 CipherConf 配置不同模块使用的加密算法，其中加密相关的模块包括了如下。

conf 对称加解密，包括了配置文件、数据库中的 config 表(包括了host key)
host 对称加解密，用于主机的登录信息加密，主要是数据库中的 host_login 表
user 非对称加解密，用于保存用户的密码信息
aksk 对称加解密，用于HTTP中的SK解密

具体的加密算法包括版本信息就通过 加密算法+版本 唯一确定。

禁止登陆 #

可以配置版本、客户端等禁止某些特征的用户登录，提供了对版本和主机的限制策略，包括了：A) 固定策略，可以设置允许/禁止的版本信息以及主机信息；B) 动态策略，在失败N次之后直接禁止其登陆。

示例配置如下。

[forbid]
VersionDeny=^SSH-1.*      # 拒绝版本的正则表达式
VersionAllow=.*OpenSSH.*  # 允许版本的正则表达式
HealthAllow=              # 只支持IP白名单设置
HostDeny=
HostAllow=cidr:192.168.9.0/24,ip:192.168.9.11   可以指定域或者IP地址
Dynamic=max-retry:in-N-seconds:prohibit-time:max-items

检查的优先级为 VersionDeny>VersionAllow>HostDeny>HostAllow>Dynamic，也就是说，配置的策略禁止的优先级要更高，如果满足禁止条件，那么直接退出；不满足则检查是否允许，不允许则直接退出。

另外，健康检查只允许设置主机允许名单，而且只能是 healthcheck 用户，只能通过用户名密码登陆，可以设置允许的主机地址，默认只能通过本地登录。

Version #

SSH 的版本信息格式为 SSH-{Version}-{ClientVersion} 的方式，例如 SSH-2.0-OpenSSH_8.8，有些客户端还会带上 OS 的版本信息，用户可以设置允许检查的正则表达式，包括 SSH 协议的版本以及用户的版本。

Host #

可以设置 IP 地址或者子网，通过逗号分割。

鉴权 #

可以控制是否开启鉴权，主要是用于测试。

[auth]
Enabled = true
Region = cn-north
Method = [role|policy|auto]

其中 Method 用来配置使用何种鉴权方式，包括了：A) Role 使用角色管理；B) Policy 使用策略管理；C) Auto 同时支持两种方式，会根据用户配置选择，优先使用 Role 策略。

如果是使用 Policy 策略，可以通过 Region 配置，其允许的正则表达式为 ^[-._a-zA-Z0-9]{1,50}$。

Trace #

支持 OpenTelemetry 格式的 Trace 实现，可以配置上报的目的地。

[trace]

其它 #

超时设置 #

# 设置会话的最长时间，超过之后会强制退出
MaxTimeout = 48h
# 鉴权最长时间，防止客户一直等待不输入密码占用资源
AuthTimeout = 5m
# 最长空闲时间，超过限制后同样退出
IdleTimeout = 30m

审计日志 #

AuditType = logger

默认是不保存审计日志的，当前也只支持保存到日志中。
