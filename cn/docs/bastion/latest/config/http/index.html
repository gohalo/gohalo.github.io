简介 #

用于配置与 HTTP 相关的内容，该模块为可选。

[http]
Enabled = True                       # 是否开启

调试 #

为了方便调试，可以通过 X-Request-Id 请求头设置 TraceID 信息，否则会自动生成随机 TraceID 信息。另外，还可以通过如下方式设置打印请求和响应信息。

DumpBody = False                     # 是否打印请求消息体
DumpBodyWithHeader = X-Request-Dump  # 通过请求头判断是否打印

注意，通过 DumpBodyWithHeader 指定的头并不判断值，只要存在头就会打印；而且只有当 Content-Type 为 application/json 时，才会打印请求体，最大长度不会超过 4096 字节。

如下是打印的日志信息。

========== HTTP Body Dump BEGIN ==========
GET /api/bastion/v1/user HTTP/1.1
Host: 127.0.0.1:9080
Accept-Encoding: gzip
Content-Type: application/json
X-Request-Dump: true
Request:
Response: {"code":0,"message":"success"}
========== HTTP Body Dump -END- ==========

Audit #

用来配置审计日志，会通过单独文件保存。

AuditPath = /var/log/cargo/bastion/audit/http.log # 审计日志路径
AuditRotate = 50,4,3,true        # 日志切割
AuditContent =                   # 审计日志格式内容

其中的日志切割可以参考 配置文件 中日志配置相关的介绍；审计日志格式可以参考 Echo Logger 中的内容。

Trace #

支持 OpenTelemetry 格式的 Trace 实现，同时需要配合基础的 Trace 配置使用。

TraceService = Bastion   # 配置服务信息
TraceVersion = 1.2.0     # 配置版本信息

如上参数用来配置 Instrumentation 相关内容，包括了 Scope Library 相关参数。

Token #

支持 JWT Token 实现的访问控制，其中密钥必须要进行配置，可以是明文或者加密。

JWTSigningMethod = HS256             # 签名方法
JWTSigningKey = YourPlainSinginKey   # 明文签名密钥，如下是密文配置
JWTSigningKeySecret = 2$0$b89QsvHP1GmwjazdAYybMiLN/TgrdjptW7qsSHM
JWTExpiry = 2h                       # 密钥过期时间，不能小于30分钟
JWTExpiryRemain = 0                  # 剩余时间小于该值影响添加 X-Token-Remain 头
JWTMaxAge = 0                        # 最大有效时间，不能超过15天

当过期时间小于 JWTExpiryRemain 阈值时，会在后续的请求中设置 X-Token-Remain 头，其值是剩余的过期时间，单位是秒，可以用于页面的鉴权的自动续期。

CROS #

也就是跨域访问配置，详细可以参考 安全配置 中的介绍。

CORSAllowOrigins = *
CORSAllowMethods = GET,HEAD,PUT,PATCH,POST,DELETE
CORSExposeHeaders = X-Token-Remain
CORSMaxAge = 0
