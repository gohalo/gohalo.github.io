用户和用户组都有各自的配置，但是用户组的优先级要更高一些，主要是为了安全考虑，用户组的配置可以覆盖个人的配置信息。而主机和主机组中同样有配置，不过此时主机的优先级更高一些，可以将用户组视为默认配置，而主机中则是不同主机的配置信息。

用户配置 #

对应 user 表中的 config 字段，详见 model.UserConfig 配置项。

{
    "login": "hosts",     # 通过shell登录时直接显示主机列表 TODO
    "shell": {
        "user_group": "your default user group", # 默认登录的用户组
        "host_group": "your default host group", # 默认登录的主机组
    },
    "config": {  # 与如下的用户组配置重叠
        "record": {
            "path": "record",   # 保存目录，保存在RecordBasePath相对路径下
            "type": ""          # 保存的类型
        }
    }
}

用户组配置 #

{
    "config": {
        "record": {
            "path": "/var/log/record",
            "type": ""
        }
    }
}

主机登录配置 #

对应 host_login 表中的 config 字段。

{
    "steps": [{   # 用于该账户的自动登录，目前只支持 Telnet
        "expect": "login:",
        "send": "{{user}}",
    }, {
        "expect": "Password:",
        "send": "{{password}}",
    }, {
        "expect": "$",
        "failed": "Login incorrect",
    }]
}

登录用户 #

可以通过配置设置登录主机时的用户，通过 user_group 中的 login_policy 参数配置，支持如下的参数配置：

  • usergroup 读取 user_group.login 字段，指定统一的登录用户。
  • user 读取 user.name 字段，由用户指定。
  • uid 读取 user.uid 字段，通过 UID 登录，适用于强制设置用户。

如果字段都是空，那么会直接返回配置文件中的 Login 字段，如果还是空，那么需要在登录时指定。详细可以查看代码中 StartChannel() 的实现。

这样可以支持如下几种场景的配置。

  • 用户配置
  • 用户组配置
  • 主机登录配置
  • 登录用户
