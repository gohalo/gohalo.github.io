API #

简单通过角色进行管理，有如下约束：

  1. 普通用户登录之后才可以修改密码、个人信息等。
  2. 管理员可以查看、修改其它用户基本信息(不含密码)。

用户登陆通过 JWT 实现，在调用 API 接口时需要在头 Authorization 中添加 Bearer TOKEN 进行鉴权，测试时可以使用内置的用户。

----- 用户注册
POST /api/auth/v1/user/register
{
    "username":"george",
    "password":"Secret"
}

----- 用户登陆
POST /api/auth/v1/user/login
{
    "username":"george",
    "password":"Secret"
}

----- 获取当前登陆用户信息，需要登陆
GET  /api/auth/v1/user/current
{
    "uid":"g001003",
    "alias":"George Washington",
    "username":"george"
}

----- 修改指定用户信息 admin
POST /api/auth/v1/user/profile
{
    "uid":"g001003",
    "alias":"George Washington",
    "username":"george"
}

----- 获取指定用户、所有用户信息 admin
GET  /api/auth/v1/user?uid=g00000001
GET  /api/auth/v1/user?offset=0&limit=10

测试命令 #

----- 登陆，通过UID和密码进行登陆，因为用户名可能不是唯一
$ curl -X POST http://127.1:1323/api/v1/login -H 'Content-Type: application/json' \
        -d '{"username":"g001003","password":"Secret"}'
{"roles":["admin","readonly"],"status":0,"token":"XXXXXXX"}

----- 查看当前用户信息
$ curl -X GET -i -H 'Authorization: Bearer TOKEN' http://127.0.0.1:1323/api/v1/user/current
{"uid":"g001003","name":"George Washington","username":"george"}

配置 #

相关的配置都保存在 configs/config.go 文件中，其中相关的配置列举如下。

JWTAlgorithm:  "HS256",                # JWT的加密方式
JWTKeyString:  "Your Key For HS256",   # 私钥
JWTKeyName:    "_user",                # 在echo传递上下文中的key
JWTExpiry:     3 * time.Hour,          # 过期时间
Backend:       InMemory,               # 角色和用户信息保存在内存中

内置用户角色 #

RESTFul API #

----- POST /api/v1/login  登陆
{
	"username": "your name",      用户名
	"password": "your password",  密码
	"type": "account",            登陆方式account(帐号)
}

----- GET  /api/v1/user/current  获取当前用户信息
{
	"id":"00000001",
	"name":"Andy Dufresne",
	"phone":"0752-268888888"
	"avatar":"https://gw.alipayobjects.com/zos/rmsportal/BiazfanxmamNRoxxVxka.png",
	"email":"antdesign@alipay.com",
	"signature":"海纳百川，有容乃大",
	"title":"高级SRE",
	"group":"某某某事业群－某某平台部－某某技术部－SRE",
	"country": "China",
	"geographic": {
		"province": {
			"key":"330000"
			"label":"浙江省",
		},
		"city":  {
			"key":"330100"
			"label":"杭州市",
		}
	},
	"address":"西湖区工专路 77 号",
	"notifyCount":12,
}

----- POST /api/v1/register  注册用户
{
	"type": "account",                 # 可选
	"name": "Foo Bar",                 # 必选，登陆用户名
	"password": "Your PassWord",       # 必选，登陆密码
}

----- GET  /api/v1/sso/user?name=FooBar 用户信息查询
{
	"name": "Foo Bar",                 # 用户名
	"roles": [ "admin", "readonly" ]   # 用户角色
}

如果 POST 请求中不包含用户名密码，但是 JWT-Cookie 是合法的，那么会自动刷新。
