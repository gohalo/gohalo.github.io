这里包括了 Web 访问页面，也有 REST API 相关的接口，而且，不需要进行鉴权。

1. WEB #

根路径是通过 html/template 渲染的模板，可以直接访问 http://localhost:1323，当前仅输出 Hello World ，后面可以修改。

2. REST API #

2.1 /rest/v1/hello #

会接受 name 和 email 两个参数，实际使用的就只有 name 参数，也就是返回 Hello <name>(Query|Form) 类似的内容，其中括号中的是接收参数方式。

----- 发送GET请求
$ curl -X GET 'http://127.1:1323/api/hello/v1/hi?name=Andy&email=andy@foobar.com'
Hello Andy(Query)!

----- 发送POST请求，使用Form格式
$ curl -X POST http://127.1:1323/api/hello/v1/hi -d 'name=Andy' -d 'email=andy@foobar.com'
Hello Andy(Form)!

----- 发送POST请求，使用JSON格式
$ curl -X POST http://127.1:1323/api/hello/v1/hi -H 'Content-Type: application/json' \
        -d '{"name":"Andy","email":"andy@foobar.com"}'
Hello Andy(JSON)!

----- 返回报错信息
$ curl -X POST http://127.1:1323/api/hello/v1/hi
{"code":150016,"message":"invalid request","cause":"invalid arguments"}

实现时，会通过 echo.Bind() 函数直接绑定参数。
