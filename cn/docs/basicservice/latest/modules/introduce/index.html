通过 Echo 框架搭建的服务端测试工具，其实也作为 GoLang 项目的一个试验田，类似于 JSONPlaceHolder 提供的 REST API 请求，不过这里提供的不只是 REST 接口，当然，也包括了一些测试功能。

1. 简介 #

1.1 目录结构 #

详细可以参考 Standard Go Project Layout 中的介绍，这里以动态库插件的形式进行模块的开发。

build/           编译二进制文件使用脚本
cmd/             可执行二进制的入口文件
  - ResTest/     对应主进程二进制文件
configs/         以GoLang源码的方式保存了一些配置信息，包括配置文件解析
internal/        真正的实现代码
  - implement/   具体插件机制的实现
    - cert/      证书管理
    - hello/     最简单的HelloWorld实现
    - login/     登陆以及用户查看
  - middleware/  Echo中间件的实现
    - logger/    日志，通过ZAP包封装的自定义实现
  - model/       各种数据模型
pkg/             一些标准包，通常可作为三方包使用
  - cert/        证书相关
scripts/         常用脚本，包括测试脚本

  template  utils

1.2 编译 #

通过 build/build 脚本编译即可，会生成 bin/ResTest 二进制文件，启动之后会监听 1323 端口。

2. 设计规范 #

2.1 路由规则 #

目前大部分提供的是 REST API 接口，其 URI 的前缀为 /rest，然后紧跟不同的服务，例如用户权限管理的为 auth，因为涉及到不同的版本，那么可以由不同的组件进行维护，所以，一个用户的查询接口大致如下。

GET /rest/auth/v1/user?uid=g00000001

3. 基础框架 #

在原有的 echo 框架的基础上进行了扩展，可以涵盖一些常见的场景。

3.1 插件功能 #

各个不同的模块通过动态库的方式进行加载，因为 GoLang 目前还不支持卸载，所以如果插件修改了，那么就需要重启服务，对于无状态的 HTTP 服务来说应该是可以接受的。

3.2 配置文件 #

程序的入口就提供了两个参数，分别是通过 -config 指定配置文件以及通过 -level 指定日志级别，也就意味着所有其它的配置项只能通过配置文件修改，否则就使用默认值。

如上所述，可以通过 -config=config.ini 参数指定，默认就是当前目录下的 config.ini 文件。

3.3 日志打印 #

其中 echo 已经提供了 e.Use(middleware.Logger()) 日志插件，同时包含了默认的日志库，但希望可以满足如下条件：

  • 保持原生的日志记录方式。
  • 日志文件支持自动切割，设置保存时长。
  • 审计日志和业务日志分开保存。

提供了一个基础插件，包含了默认日志模块，可以通过 logger.Info() 类似的方式打印，用户也可以创建自定义的日志。

为了方便跟踪，会通过 middleware.RequestID() 插件生成请求 ID 标识，保存在 X-Request-ID 头中，如果请求携带了该头文件，那么就不会更新。

3.4 异常处理 #

提供了统一的处理函数 APPErrorHandle()，当前返回的为 json 格式。

{
    "Code": 0,                         统一错误码
    "Cause": "invalid parameters",     统一错误信息，是错误码的解释
    "Detail": "failed to login",       具体的错误信息
}

如果有明确的异常，可以在 echo 处理函数中通过 utils.NewAPPError(utils.EcodeAuthenticated, "detail error message") 返回，其中的错误码对应了错误信息。

3.6 国际化 #

会以此从 A) HTTP 头 Accept-Language；B) Cookie 中的 lang；C) 参数列表中的 lang 。与翻译相关的保存在 internal/i18n 目录下。

4. 功能模块 #

这里的很多功能都是通过动态库插件实现，在启动时会调用 LoadModule(可选)、RegisterHandle(必选) 函数进行初始化以及注册，退出时会调用 UnloadModule(可选) 。

  • [基本功能]({{x relref “/book/devops/utils/restest/hello” >}}) 不需要鉴权，类似于 HelloWorld ，可以发送一些基本请求，包括了初始 Web 页面。
  • [用户权限]({{x relref “/book/devops/utils/restest/authority” >}}) 这里包含了登陆系统、用户管理、角色管理、权限管理，也就是授权相关的关键内容。
  • [证书管理]({{x relref “/book/devops/utils/restest/certification” >}}) 作为证书管理，包含了整套的 PKI 体系，可以用来创建根证书、中间证书、服务器证书等。

这里暂时仅采用 JWT Token 机制，并没有使用 Session 机制。
