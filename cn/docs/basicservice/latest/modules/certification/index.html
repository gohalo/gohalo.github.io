1. 简介 #

私钥加密算法，用来对私钥进行加密，采用的是非对称加密算法，会同时生成公私密钥对，可选的加密算法有：

RSA-2048     RSA加密，同时指定长度
ECDSA        包括了P224 P256 P384 P521几种
Ed25519

1.1 证书名称 #

证书会按照树结构进行展示，显示时，会按照一定规则选择名称，对于页面来说，主要是为了方便查看，查找顺序为：

  • 名称，为了方便识别，允许用户设置 name 字段，该字段仅用于标识证书，方便查看。
  • CommonName 一般是域名或者使用通配符的泛域名。
  • SerialNumber 通常是在创建证书时随机生成的。

1.2 最佳实践 #

这里列举创建证书时常见的配置项。

Root  有效期可以为20年
  RSA-2048
  KeyUsage --> Certificate Signing + CRL Signing 允许签发、撤销证书
  BasicConstraints --> Certificate Authority(Yes) 允许签发证书
Middle 有效期一般为10年
  RSA-2048
  KeyUsage --> Certificate Signing + CRL Signing 允许签发、撤销证书
  BasicConstraints --> Certificate Authority(Yes) 允许签发证书
  CRL Endpoints 用来下载CRL列表
  Authority Info (AIA) 一般也就是OSCP
  Certificate Policies 可以参考 https://www.globalsign.com/en/repository 不清楚是干啥的
Server 有效期一般为一到两年
  RSA-2048
  KeyUsage --> Digital Signature, Key Encipherment 用于签名以及加密
  BasicConstraints --> Certificate Authority(No) 不允许签发证书

2. API #

会保存公钥以及私钥，所以，建议对私钥进行加密，这里不会保存私钥。

----- 创建证书，包括了根证书、中间证书、服务器证书
POST /api/v1/cert
{
    "Name": "DevOps Agent",             可选，为方便查看证书
    "Type": "Root",                     必选，证书类型 Root Middle Server
    "Private": {                        私钥信息
        "Method": "RSA",                私钥加密算法，目前支持RSA
        "Parameter": "1024",            加密算法参数，对于RSA表示加密位数
    }
    "Certificate": {
        "Parent": {
            "SerialNumber": "HHHHHHH",             对 Middle Server 两类来说，必须指定签发证书
        },
        "NotAfter": "2021-02-10T22:27:53+08:00",   证书的开始时间
        "NotBefore": "2021-02-10T22:27:53+08:00",  证书的结束时间
        "Subject": {
            "C": "CN",
            "CN": "baidu.com",
        }
    }
}

----- 查看/下载证书、私钥
GET /api/v1/cert?cn=
GET /api/v1/cert?id=
GET /api/v1/cert?name=
GET /api/v1/cert?serial=

----- 撤销证书
DELETE /api/v1/cert

----- OCSP
/api/v1/ocsp


curl -X GET 'http://127.1:1323/api/v1/cert/info?id=1'

3. 表结构设计 #

CREATE DATABASE IF NOT EXISTS `CertServer`;
USE `CertServer`;

DROP TABLE IF EXISTS `certificate`;
CREATE TABLE IF NOT EXISTS `certificate` (
    `id` INT NOT NULL AUTO_INCREMENT PRIMARY KEY,
    `parent` INT NOT NULL DEFAULT -1 COMMENT "父证书的ID信息，用来构建层级关系",
    `type` INT NOT NULL COMMENT "证书类型：1-根CA证书，2-中间证书，3-服务器证书",
    `serial_number` CHAR(64) NOT NULL COMMENT "证书的SerialNumber十六进制表示",
    `name` VARCHAR(128) NULL COMMENT "用户自定义的证书名称",
    `common_name` VARCHAR(128) NOT NULL COMMENT "证书的CommonName信息",
    `private` VARBINARY(4096) NOT NULL COMMENT "私钥信息 DER 格式",
    `certificate` VARBINARY(4096) NOT NULL COMMENT "证书内容 DER 格式",
    `gmt_modify` TIMESTAMP NOT NULL ON UPDATE CURRENT_TIMESTAMP,
    `gmt_create` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT "保存证书信息";

4. 测试用例 #

这里假设一个场景，在中国 (C) 浙江 (ST) 杭州 (L) 有一个名为 Cargo (O) 的公司，其中 CertCenter (OU) 部门负责管理证书，公司下的域名为 *.cargo.com ，那么可以设置三级证书。

----- 根证书，其Subject Name和Issuer Name相同，CN一般非域名
C=CN,O=Cargo,OU=CertCenter,CN=Cargo Trust Root R1

----- 中间证书
C=CN,O=Cargo, Inc.,CN=Cargo Trust CA-1

----- 服务器证书，服务器最终使用
C=CN,ST=ZheJiang,L=HangZhou,O=Cargo, Inc.,CN=*.cargo.com
C=CN,ST=ZheJiang,L=HangZhou,O=Cargo, Inc.,CN=www.cargo.com

脚本详见源码中 scripts/cert/gencert.sh 的实现，会生成如下文件。

pki
|-ca/              ----- 证书链，最上一层是根证书

脚本 #

非对称密钥对 #

包括了 RSA-1024、RSA-2048、RSA-4096、ED25519、ECDSA-P224、ECDSA-P256、ECDSA-P384、ECDSA-P512。

注意，目前只支持 RSA 算法，其它没有验证过。

----- #1 RSA 生成、查看未加密的秘钥文件
openssl genrsa -out private.pem 1024
openssl rsa -in private.pem
openssl rsa -in private.pem -text -noout
PrivateKey{Method: "RSA", Parameter: "1024"}

----- #2 ECDSA 生成、查看未加密的秘钥文件
openssl ecparam -list_curves    # 支持的ECDSA算法
openssl ecparam -name secp384r1 -genkey -out private.pem

加密私钥 #

包括了 DES、3DES、AES128、AES192、AES256 几种。

------ 对秘钥加密
openssl genrsa -out private_des.pem -des3 -passout 'pass:YourPassHere' 1024
openssl rsa -in private_des.pem -text -noout -passin 'pass:YourPassHere'
PrivateKey{Method: "RSA", Parameter: "1024"}
Dump("key.pem", "YourPassHere", "DES3")

------ 检查秘钥文件是否合法
openssl rsa -in private.pem -check
