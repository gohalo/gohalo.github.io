1. 常用命令 #

1.1 代码同步 #

这里的通用库采用 git 的 submodule 方式添加到各个项目中，以 liblog 库为例，大致的使用方式如下。

----- 以submodule方式添加库
$ git submodule add git@gogs.cargo.com:cargo/mote-liblog.git libs/liblog

----- 对于所有的submodule拉取最新的代码
$ git submodule foreach --recursive git pull
$ git submodule update --init --recursive

----- 可以简单查看当前各个模块的状态
$ git submodule

----- 删除模块
$ git rm --cached libs/liblog

1.2 编译选项 #

在项目中通过 cmake .. -DCMAKE_BUILD_TYPE=Debug -DWITH_EXAMPLES=true 类似的方式进行编译，主要有如下的几个选项：

  • -DCMAKE_BUILD_TYPE=Debug 指定编译模式。
  • -DWITH_EXAMPLES=true 编译代码中的示例。
  • -DWITH_UNIT_TESTS=true 编译单元测试代码。
  • -DWITH_CONVERAGE=true 生成代码覆盖率文件。

总的命令为。

cmake .. -DCMAKE_BUILD_TYPE=Debug -DWITH_EXAMPLES=true -DWITH_UNIT_TESTS=true -DWITH_CONVERAGE=true

目前的库正在调整支持单独编译，部分库可能仍须要在某个项目中编译。

1.3 代码覆盖率 #

生成代码覆盖率的 CMake 模块一般会保存在 contrib/cmake 目录下，可以通过如下方式使用。

IF(WITH_CONVERAGE)
    SET(CMAKE_MODULE_PATH ${CMAKE_SOURCE_DIR}/contrib/cmake)

    SET(CMAKE_C_FLAGS_DEBUG   "${CMAKE_C_FLAGS_DEBUG} -fprofile-arcs --coverage")
    SET(CMAKE_CXX_FLAGS_DEBUG "${CMAKE_CXX_FLAGS_DEBUG} -fprofile-arcs --coverage")

    INCLUDE(CodeCoverage)
    SET(COVERAGE_EXCLUDES "'*mock*'" "'*tests*'")
    SETUP_TARGET_FOR_COVERAGE(coverage ctest covdir)
ENDIF()

注意，编译如果是 -DCMAKE_BUILD_TYPE=Debug 需要添加上述的 C_FLAGS 参数，也可以使用 -DCMAKE_BUILD_TYPE=Coverage 编译，此时在插件里会设置相应的参数，从而可以省略上述参数的配置。

源码可以从 CodeCoverage.cmake 下载，通过如下命令查看代码覆盖率，访问 localhost:8080 即可以看到报告。

make coverage
python -m SimpleHTTPServer 8080
python3 -m http.server -d covdir 8080
