简介 #

在启动时可以通过命令行和配置文件修改默认配置，绝大部分都可以通过命令行配置，而且命令行的优先级更高。

目前所有的配置保存在 daemon/config/config.c 文件中的全局变量中，包含了用户的配置以及默认配置，只有有限几个参数是可以通过服务端动态修改的，大部分只允许启动时通过命令行或者配置文件配置。

因为 BootAgent 可以动态修改的配置项是通过文件保存，为了保证配置修改的原子性，提供了配置的 2PC 操作 (没有默认值)，实现时是通过文件系统的原子移动完成的。

配置项 #

其中配置文件也就是 BootAgent.conf 文件，该文件只读，也就意味着在安装时已经配置好，其中，部分参数可以编译阶段通过宏定义，括号中的值是默认值，注意，是生产环境的值，测试阶段一般是相对路径。

文件采用 ini 格式，如下配置在 | 之后为命令行参数。

umask(027)
   进程 umask 设置
daemonize(true | -f)
   是否后台运行
log.file(/var/log/cargo/BootAgent.log | -L arg)
   日志文件路径
log.level(info | -l arg)
   日志级别
base.dir(/usr/local/cargo/BootAgent)
   运行时通过chdir切换目录

原子操作 #

部分配置项支持原子操作，主要是通过文件系统的原子移动来实现，如果已经存在，那么就需要先备份才可以，当更新失败，就可以直接回滚。而在读取时，如果发现文件不存在，那么还需要检查备份的文件是否存在，如果存在，那么就需要直接读取。

命令行参数 #

这里包含了几种参数。

BootAgent #

这个是主要的进程，一部分常用的参数可以通过命令行配置。

-f
    前台运行，默认是后台运行。
-l <level>
    日志级别，可以
-L <path>
    日志保存路径。
-P <path>
    PIDFile 保存路径。
-S <list>
    服务器地址。
-C <name>
    cgroup 组名称。
-T
    用于测试当前的配置，而非真正运行，为了防止覆盖老日志，可以同时使用 -L 参数。

BootAgentCtrl #

一般用来调试、检查状态等使用，可以通过 BOOTAGENT_TRACE 环境变量来查看 bootctl 与 bootagent 的命令交互过程。

----- 设置日志级别
export BOOTAGENT_TARCE=1; daemon/BootAgentCtrl config setlog trace

----- 检查文件 SHA256
daemon/BootAgentCtrl sha256 <YourFile>

----- 下载文件
daemon/BootAgentCtrl download <FilePath>

Systemd #

在 CentOS 中也就是通过 systemd 管理进程，其中相关的 service 文件可以参考 contrib/BootAgent.service 中的配置内容。

----- 更新配置
# cp BootAgent.service /usr/lib/systemd/system/BootAgent.service
# systemctl daemon-reload

----- 查看状态以及重启
# systemctl status BootAgent
# systemctl restart BootAgent

----- 配置后的cgroup会添加到如下目录
$ ls /sys/fs/cgroup/cpu/system.slice/BootAgent.service
$ ls /sys/fs/cgroup/memory/system.slice/BootAgent.service
